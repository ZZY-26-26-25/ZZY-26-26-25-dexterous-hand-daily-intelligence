## 2. ModPack 软件冒烟测试

### 目标

验证可穿戴遥操作系统是否能将手臂、底盘、头部主动视角、力反馈和相机流作为独立模块接入统一 logger。

### 步骤

1. 拉取 `citronCode/modpack`，固定 Git commit 与依赖版本。
2. 审阅 ARX5 和 RB-Y1m 配置，整理所有模块名称、topic/stream、频率和坐标系。
3. 在无真机模式下构造模拟流：
   - `leader_joint_state`
   - `follower_joint_state`
   - `operator_head_pose`
   - `operator_base_pose`
   - `camera_stream`
   - `ft_wrench`
   - `haptic_torque_command`
4. 录制一个 60 秒模拟 episode，验证 pause/resume、丢帧、时钟偏差和文件完整性。
5. 将日志转换为 MCAP 或现有统一格式，检查每条流是否保留原始时间戳。
6. 若软件链路稳定，再进入单臂主设备阶段；最后再扩展双臂、移动底盘与高 DoF 手套。

### 必须记录

- `module_manifest`
- `module_role`
- `clock_domain`
- `stream_name`
- `nominal_rate_hz`
- `measured_rate_hz`
- `host_timestamp`
- `device_timestamp`
- `drop_count`
- `operator_head_pose`
- `operator_base_pose`
- `leader_joint_state`
- `follower_joint_state`
- `ft_wrench`
- `haptic_torque_command`

### 验收指标

- 60 秒记录无结构损坏
- 所有流可回放
- 相机、关节与力觉时间偏差可统计
- 模块可单独启停而不破坏其他流
- 更换手指模块时无需修改手臂/底盘 logger

## 3. EgoRecovery 最小复现

### 目标

将失败与恢复建立为一级数据类型，验证人类第一视角恢复片段是否能以较低成本补充机器人恢复数据。

### 任务建议

优先选择 2–4 个能稳定重置的桌面任务：

- 插入后偏斜
- 抓取后滑落
- 放置位置偏移
- 叠放后倾倒
- 清扫时遗漏目标区域

### 数据采集

1. 为每项任务定义 5–10 种 `off_nominal_type`。
2. 人类端只录制从失败状态到恢复完成的短视频。
3. 机器人端采少量相同失败状态的恢复示教。
4. 对每段数据标记：
   - 失败开始帧
   - 恢复开始帧
   - 恢复结束帧
   - 纠错方向与幅度
   - 是否恢复成功
5. 建立 robot-only、direct mixing、gated recovery 三种基线。

### 必须保存

- `failure_state_id`
- `off_nominal_type`
- `failure_onset_frame`
- `recovery_start_frame`
- `recovery_end_frame`
- `corrective_intent`
- `recovery_gate`
- `recovery_success`
- `human_or_robot_source`
- `collection_minutes`
- `validity_reason`

### 评价指标

- initial success rate
- recovery success rate
- final task success rate
- false recovery trigger rate
- recovery latency
- 每小时有效恢复片段数
- 人类数据替代机器人数据的成本比

## 4. HOST 数据层复现

### 目标

在正式代码发布前，先让平台具备“单个参考视频—任务进度—未来观察—机器人动作”的数据表示能力。

### 建议流程

1. 选择一个已有机器人任务及一段对应人类视频。
2. 将两者切成语义阶段，并为每帧标注归一化 `task_progress`。
3. 对齐人类视频与机器人轨迹的阶段边界，而不是直接对齐关节动作。
4. 生成若干未来观察目标或关键帧。
5. 保存技能检索键和数据来源，确保后续可替换模型。

### 必须保存

- `source_video_id`
- `source_video_type`
- `language_instruction`
- `task_progress`
- `progress_alignment_version`
- `future_observation_target`
- `skill_memory_key`
- `robot_episode_id`
- `acquisition_time_seconds`
- `parameter_update`

### 限制

在代码、权重和数据开放前，不应声称完成 HOST 方法复现；当前工作仅是建立兼容的数据结构和评测接口。
