# 灵巧手每日情报｜2026-07-24

## 0. 今日一句话判断

过去 24–72 小时的高价值新增主要集中在四条线上：**多本体语义抓取数据、脆弱物体抓力预测、跨本体像素动作世界模型，以及纯机械欠驱动灵巧手设计**。其中，SeededGrasp 首次把 Franka 夹爪、Allegro Hand 和 Robotiq 三指手纳入同一杂乱场景抓取数据库；Masked Visual Actions 则把机器人或物体在视频中的局部运动直接作为动作表示，并已经公开 LoRA 权重与训练/推理代码。

今天没有发现过去 24–72 小时内可核验的全新商业灵巧手产品、标准报价或正式 SDK 大版本发布。与我们的“灵巧手数据采集平台”最相关的结论是：**同一条人类或机器人演示应同时保留语义目标、空间 seed、目标手 embodiment、物体运动、机器人动作和接触/抓力结果，避免把所有任务压缩成单一关节轨迹。**

## 2. 重点解读

### SeededGrasp

- **原始链接：** [arXiv](https://arxiv.org/abs/2607.20207) · [项目页](https://uoft-isl.github.io/seeded-grasp/) · [GitHub](https://github.com/uoft-isl/SeededGrasp)
- **一句话概括：** 先让 VLM 根据语言在 RGB-D 图像上选择一个语义 seed，再由轻量 flow-matching 模型生成目标手可执行的 6D 抓取位姿。
- **硬件是什么：** 训练数据库覆盖 Franka Panda 夹爪、Allegro Hand 和 Robotiq 3-Finger；真机实验使用 Delto DG-3F-B，并将 Robotiq 三指手条件下的输出映射到 Delto。
- **采集/控制链路：** 文本任务 → 预训练 VLM 输出 2D seed → RGB-D 投影为 3D 点 → embodiment-conditioned flow matching → 6D 抓取位姿 → 仿真/真机执行。
- **自由度与传感：** 数据库不是手套关节数据，而是 610 个杂乱桌面场景中的 2.56M 个稳定抓取位姿；含 334 个 GSO/YCB 物体，其中 50 个物体仅用于未见评估。
- **反馈方式：** 主要以仿真与真机抓取成功率评价，不使用触觉或抓力闭环。
- **方法核心和实验：** 数据量按 embodiment 分为 Franka 约 2.04M、Allegro 约 814K、Robotiq 三指手约 273K（训练与评估合计）；整体仿真成功率 72%，Delto 真机实验 78%。语义推理与几何执行解耦，避免从头联合训练 VLM 与抓取网络。
- **和 DexUMI / DOGlove / UMI-Dex / SynGlove 的关系：** 这些方案提供人类示教或机器人动作；SeededGrasp更适合作为抓取初始化器或数据扩增器。未来可以用手套/UMI记录“人选择了哪个物体、从哪里接近”，再用 SeededGrasp 生成适配不同目标手的候选抓取。
- **对我们搭建平台的启发：** 新增 `language_instruction`、`target_object_id`、`seed_pixel`、`seed_point_3d`、`embodiment_id`、`grasp_pose_6d`、`grasp_quality` 和 `execution_success`。同一场景应允许保存多种手型的候选抓取。
- **风险或不确定性：** 项目页称公开代码与数据库，但截至本次检查 GitHub 只有 README 和“Code and database coming soon”；真机使用不同三指手之间的映射，尚不能证明对五指高自由度手具有同等迁移效果；缺少触觉与抓力安全约束。

### V2F：Vision-Informed Grasp Force Prediction

- **原始链接：** [arXiv](https://arxiv.org/abs/2607.19804) · [HTML](https://arxiv.org/html/2607.19804)
- **一句话概括：** 在真正接触脆弱水果之前，仅通过外观、几何和品种信息预测一个既不会滑落、又不会造成压伤的抓取力。
- **硬件是什么：** 数据采集使用 UR10e、ZED 双目相机、定制亚克力夹持面与高精度 load cell；操作验证使用 Robotiq 2F-85 和负载传感器。
- **采集/控制链路：** RGB/双目图像 → Mask R-CNN 与主动轮廓精修 → 尺寸和形状描述 → Hertz 接触模型给出物理基线 → 残差神经网络根据品种与视觉特征修正 → 生成安全抓力设定值。
- **自由度与传感：** 研究对象不是灵巧五指手，而是夹爪抓力；约 500 个样本覆盖 Ajwa、Barhi、Sagai 三种椰枣及正常、湿润、干燥状态。
- **反馈方式：** 负载传感器用于标定与验证；部署目标是减少或避免缓慢、可能损伤物体的触觉探测。
- **方法核心和实验：** 将 Hertz 接触解析模型与数据驱动残差结合，在未见品种组上达到约 R²=0.7；预测抓力约在 0.3–1.25 N 范围内，真机抓放中未观察到滑移或表面损伤，残余形变约 0.4–0.8 mm。
- **和现有方案的关系：** Current as Touch、NeuralActuator、Revo3/Wuji 触觉接口是在接触后估计力；V2F属于接触前预测。两者可组合成“视觉先验抓力 + 触觉/电流在线修正”。
- **对我们搭建平台的启发：** 增加 `pre_contact_force_setpoint`、`visual_geometry_features`、`material_or_cultivar_id`、`force_model_version`、`measured_force`、`slip_event` 和 `damage_result`，并区分预测抓力与执行端实际力。
- **风险或不确定性：** 当前只覆盖椰枣和两指夹爪；视觉到力的映射依赖材料和含水状态，迁移到软包装、线缆、纸杯或五指手时需要重新机械标定；没有公开代码或数据。

### Masked Visual Actions

- **原始链接：** [arXiv](https://arxiv.org/abs/2607.19343) · [项目页](https://masked-visual-actions.github.io/) · [GitHub](https://github.com/HadiZayer/masked-visual-actions) · [Hugging Face 权重](https://huggingface.co/HadiZayer/masked-visual-actions)
- **一句话概括：** 不把动作写成固定机器人关节或末端向量，而是把视频中机器人或物体的一部分运动轨迹显式显示出来，让视频模型补全剩余世界变化。
- **硬件是什么：** 方法在 DROID 机器人数据、BEHAVIOR 仿真和自采真机任务上验证；项目页展示的所有测试 embodiment 均未在训练中出现。
- **采集/控制链路：** 初始真实图像 + masked action video + 文本 → Wan2.2 控制视频模型 LoRA → 预测完整 RGB rollout。Forward 模式显示机器人运动，用于评估动作后果；inverse 模式显示目标物体运动，再生成满足该结果的机器人运动。
- **自由度与传感：** 不绑定固定 DoF；动作载体可以是机器人像素、物体像素甚至从人类示教中抽取的目标物运动。只需要视频和 mask，不要求完整机器人关节标签。
- **反馈方式：** 世界模型提供视觉结果预测；不包含力觉、触觉或真实接触状态。
- **方法核心和实验：** 仅用 15 小时 masked 真实/仿真视频微调单一 checkpoint；项目页展示 4 项任务、80 条真实演示的预测与真实结果对比。相较 skeleton 或 EEF 条件，masked action 在未见夹爪和双臂 embodiment 上更稳定。
- **开源状态：** GitHub 使用 Apache-2.0，包含推理脚本、双 expert LoRA 训练 recipe 与公开权重；DROID episode 转 URDF 控制视频的渲染工具仍标记 coming soon。
- **和 DexUMI / DOGlove / UMI-Dex / SynGlove 的关系：** 这些设备产生的视频、腕部轨迹和物体操作结果可以被转换成 masked object action，避免先恢复高精度机器人关节动作。它尤其适合利用普通人类视频和跨本体数据，但不能替代高保真执行数据。
- **对我们搭建平台的启发：** 数据中增加 `masked_entity_type`、`mask_video`、`object_motion_target`、`robot_render_video`、`reference_frame`、`world_model_checkpoint` 和 `generated_rollout`。原始视频与对象 mask 必须永久保留。
- **风险或不确定性：** 基础模型为 14B，训练/推理成本高；对精细接触、微小旋转和初始画面未显示区域容易生成不自然结果；视觉上合理不等于接触力学真实。

### 被动旋转欠驱动手

- **原始链接：** [arXiv](https://arxiv.org/abs/2607.18950)
- **一句话概括：** 在每根两节指的根部加入被动旋转轴，并用跨指差动机构分配一个驱动器的力矩，使手指无需主动预摆就能自动适配圆柱、球体和平行抓取。
- **硬件是什么：** 完全机械的欠驱动原型；单指两节指骨，根部被动旋转，多个手指通过差动机构共享驱动力。
- **采集/控制链路：** 当前主要依靠人工/机械驱动展示，没有机器人学习数据采集与自动控制链路。
- **自由度与传感：** 论文重点是被动与欠驱动自由度，未配置编码器、触觉、腕部6D或主动反馈。
- **反馈方式：** 接触力通过机械结构被动重新分配；以指根接触力矩稳定性条件判断物体是否可能被挤出。
- **方法核心和实验：** 理论分析了指根被动轴上的接触力矩与抓取稳定性，并用原型展示圆柱、球形、平行和包络抓取。设计通过结构本身降低控制、执行器和成本需求。
- **和 LEAP/MIDAS/Wuji 等关系：** 它牺牲独立关节控制换取低成本自适应，与全驱动研究手不同；更适合物流、农业、垃圾分拣和辅助抓取，不适合需要逐指高精度示教复现的完整灵巧操作。
- **对我们搭建平台的启发：** 硬件数据库应分开记录 `active_dof`、`passive_dof`、`coupled_dof`、`differential_topology` 和 `passive_axis`。同样的“总DoF”不代表同样的数据采集与控制能力。
- **风险或不确定性：** 尚无自动驱动、传感器、CAD、BOM或控制代码；当前任务主要是静态抓取，不能证明手内操作、快速换指和高精度接触能力。

## 4. 商业产品与采购线索

本轮未发现过去 24–72 小时内具有完整技术文档的新商业灵巧手型号、标准价格或正式 SDK 大版本发布，因此不重复 OmniHand 3 Ultra-M、Wuji Hand 2、Revo3、mimic M1/U1、TeleHand、DAS Dex 等既有条目。

今日新增的采购判断主要是方法层面：

- **抓取执行手采购必须确认力控制接口。** SeededGrasp 只能给出抓取位姿，V2F 只能给出预测抓力；若目标手/夹爪无法读取实际电流、力、触觉或滑移，二者难以形成安全闭环。
- **跨本体数据应优先选择开放 URDF 与关节映射的手。** 多本体抓取与 masked action world model 都依赖准确的目标手模型、渲染和碰撞定义。
- **欠驱动手适合低成本抓取，不等同于灵巧数采执行端。** 采购时不能只比较“总自由度”，必须看主动、耦合和被动自由度的组成。

- **链接：** [项目页](https://uoft-isl.github.io/seeded-grasp/)

## 6. 对我们项目的直接建议

1. **把 SeededGrasp 纳入多本体抓取数据库跟踪，但等待真实数据释放后再投入复现。** 当前先建立字段和下载监控，不要把占位仓库误判为已开放数据集。
2. **新增“语义 seed → 6D抓取 → 实际力闭环”的分层记录。** 抓取位姿、预测抓力、执行电流、触觉和滑移应分别保存。
3. **拉取 Masked Visual Actions 仓库做推理 smoke test。** 先使用作者权重和一段现有机器人视频验证 mask 输入格式，不必立即训练14B模型。
4. **将 masked visual action 加入动作表示分类。** 它不同于关节空间、EEF 6D和光流，是“局部像素轨迹 + 首帧”的跨本体动作接口。
5. **软物体/易损物体任务新增 damage ground truth。** 除成功率外记录永久变形、压痕、破损、滑移和最大抓力。
6. **硬件表继续严格拆分主动、耦合和被动 DoF。** 被动旋转欠驱动手说明，20个运动自由度可能只需要很少执行器，但无法提供同等动作标签和独立控制。
7. **保留物体 mask、目标物运动和相机原始数据。** 这样未来可从普通人类视频生成 masked object action，而不依赖完整人手关节重建。
