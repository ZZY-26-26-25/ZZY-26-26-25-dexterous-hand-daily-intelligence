# 灵巧手每日情报｜2026-07-24

## 0. 今日一句话判断

过去 24–72 小时的高价值新增集中在五条线上：**多本体语义抓取、脆弱物体接触前抓力预测、单个人类视频即时技能习得、第一视角失败恢复数据，以及模块化全身遥操作硬件**。SeededGrasp 与 Masked Visual Actions 继续推动跨本体动作与抓取表示；HOST 和 EgoRecovery 则说明，大量人类视频不仅能补充成功示教，还可在推理时快速注入新技能并覆盖机器人遥操作难以规模化采集的失败恢复长尾。

今天没有发现可核验的全新商业灵巧手型号、标准报价或正式 SDK 大版本。最值得立即进入工程评估的是 ModPack：它将可穿戴计算、电源、存储、同构主臂、移动底盘、主动视角和关节级力反馈组织成可替换模块，并已公开遥操作与数据记录软件。对我们的“灵巧手数据采集平台”而言，应进一步保存**任务进度、失败状态、纠错意图、恢复门控、操作者底盘/头部运动，以及人类视频到机器人动作之间的中间表示**。

## 2. 重点解读

### HOST：单个人类视频即时技能习得

- **原始链接：** [arXiv](https://arxiv.org/abs/2607.20033) · [HTML](https://arxiv.org/html/2607.20033)
- **一句话概括：** 不对策略做任务级微调，而是在推理时让机器人持续定位自己处于人类示教的哪个阶段、预测下一阶段在机器人视角中应当出现什么，再由预测结果导出动作。
- **硬件是什么：** 论文聚焦通用视觉操作策略和人类视频，不要求操作者佩戴手套或目标机器人具有特定灵巧手；测试任务覆盖多种物体、工具和操作原语。
- **采集/控制链路：** 单个人类视频 → 人类/机器人轨迹映射到共享任务进度流形 → 机器人在线估计当前进度 → 预测机器人自身未来观察 → 从未来观察预测动作。
- **自由度与传感：** 主要依赖视觉、机器人状态和动作，不要求显式恢复完整人手关节或腕部 6D。预训练使用 193,462 条机器人轨迹、229 个任务；人类到机器人适配使用 5,847 个人类视频示教及其同任务机器人轨迹。
- **反馈方式：** 机器人闭环观察与动作反馈；没有触觉或力觉。
- **方法核心与实验：** 在 50 个未出现在训练中的新任务上，每项使用一个人类视频且不更新参数，平均成功率 62%，平均约 29 秒完成技能获取；相较最强零样本基线高 45 个百分点，并避免任务微调造成的旧技能退化。
- **和 DexUMI / DOGlove / UMI-Dex / SynGlove 的关系：** HOST 使用的视频监督比手套/UMI更廉价，但缺少接触、触觉和高精度手指动作。硬件采集数据可作为机器人动作锚点，而大量普通视频可扩展任务和场景覆盖。
- **对我们搭建平台的启发：** 增加 `source_video_id`、`task_progress`、`progress_alignment_version`、`predicted_future_observation`、`skill_memory_key`、`acquisition_time` 和 `parameter_update=false`。
- **风险或不确定性：** 平均 62% 仍不足以替代高可靠示教；强接触、精细五指动作和触觉不可见状态未被显式解决；正式代码和完整数据发布状态需继续跟踪。

### EgoRecovery：人类第一视角失败恢复数据

- **原始链接：** [arXiv](https://arxiv.org/abs/2607.19745) · [HTML](https://arxiv.org/html/2607.19745) · [GitHub](https://github.com/EgoRecovery/EgoRecovery)
- **一句话概括：** 通过预先布置失败状态，让人只录制短小的恢复动作片段，再将人类和机器人的恢复过程对齐到共享纠错意图空间。
- **硬件是什么：** 人类端使用第一视角相机采集恢复片段；机器人端在真实桌面平台执行杯刷插入、桌面清扫、圆盘放置和立方体堆叠。
- **采集/控制链路：** 设计任务级 off-nominal 状态 → 人类快速录制恢复片段 → 手部运动重建、过滤和重采样 → 学习纠错时机与幅度的低维 corrective intent → 少量机器人恢复数据完成动作接地 → recovery gate仅在失败状态激活纠错。
- **自由度与传感：** 不要求人类动作直接等同机器人关节；部署时策略只接收机器人观察，不访问人类视频或人工恢复标签。
- **反馈方式：** 失败检测、纠错意图和闭环恢复结果；不以真实触觉为核心。
- **方法核心与实验：** 人类成功/恢复片段采集约 500 episode/操作者小时，机器人约 50 episode/小时。固定 50 条机器人成功、50 条机器人恢复和 300 条人类恢复时，EgoRecovery 的 Initial SR 为 80%，Recovery SR 为 85%；直接混合相同数据均为 71.2%。
- **和现有采集路线的关系：** 传统带本体遥操作最适合采成功执行和执行端反馈，但故意制造并恢复大量失败成本高。EgoRecovery更接近低成本人类视频扩增，最终仍需少量机器人恢复数据连接到可执行动作。
- **对我们搭建平台的启发：** 失败 episode 不应删除。建议增加 `failure_state_id`、`off_nominal_type`、`recovery_window`、`corrective_intent`、`recovery_gate`、`recovery_start_frame`、`recovery_success` 和 `human_robot_cost_equivalent`。
- **风险或不确定性：** 当前四个任务的失败模式较受控；触觉、执行器卡滞和复杂多指滑移未显式覆盖；代码仓库完整性与数据许可需要核验。

### ModPack：模块化可穿戴全身遥操作底座

- **原始链接：** [arXiv](https://arxiv.org/abs/2607.19479) · [项目页](https://modpack-robotics.github.io/) · [GitHub](https://github.com/citronCode/modpack)
- **一句话概括：** 把电源、计算、通信和数据存储装入可穿戴背包，再按任务插接同构主臂、底盘追踪、主动视角和关节级力反馈模块。
- **硬件是什么：** 3D打印背包主体；ARX5平台使用一对6-DoF Dynamixel同构主臂，RB-Y1m使用一对7-DoF主臂。Vision Pro控制机器人头部相机，背包上的iPhone追踪操作者SE(2)运动以控制移动底盘。
- **采集/控制链路：** 同构主臂关节 → follower关节空间；iPhone WebXR → 底盘位姿；Vision Pro头部6D → 机器人颈部；机器人F/T → Jacobian转关节力矩 → 主臂电机反馈；所有相机、本体与命令流由背包PC统一记录。
- **自由度与传感：** 支持6或7 DoF双臂、移动底盘、头部主动视觉、多相机和机器人关节力矩。公开配置示例支持100 Hz关节流与30 Hz、1280×720相机流。
- **反馈方式：** 机器人末端F/T映射到主臂关节，形成可穿戴关节级力反馈，并叠加重力补偿和电流限幅。
- **方法核心与实验：** 布料放置任务中头部相机策略22/25成功，腕部相机仅3/25；箱体转移任务中全相机+力矩为12/20，纯全相机为6/20，说明力信号可改善抓取时机和深度判断。
- **和 DexUMI / DOGlove / UMI-Dex / SynGlove 的关系：** ModPack当前主要解决双臂、底盘和视角，不提供高DoF手指输入。它可与DOGlove、MANUS、Wuji Glove或我们的22路外骨骼并联，形成“主臂负责手腕/全身、手套负责手指”的完整系统。
- **对我们搭建平台的启发：** 控制模块应物理和软件解耦，增加 `module_manifest`、`operator_base_pose`、`operator_head_pose`、`leader_joint_state`、`haptic_torque_command`、`camera_stream_role` 和 `backpack_clock`。
- **风险或不确定性：** 本次仓库不含策略训练/推理代码；设备较重且系统复杂；Wi-Fi、CPU频率和日志负载会影响控制实时性；手指级接口需要二次集成。

### SeededGrasp

- **原始链接：** [arXiv](https://arxiv.org/abs/2607.20207) · [项目页](https://uoft-isl.github.io/seeded-grasp/) · [GitHub](https://github.com/uoft-isl/SeededGrasp)
- **一句话概括：** 先让 VLM 根据语言在 RGB-D 图像上选择语义 seed，再由轻量 flow-matching 模型生成目标手可执行的 6D 抓取位姿。
- **硬件是什么：** 训练数据库覆盖 Franka Panda、Allegro Hand 和 Robotiq 3-Finger；真机使用 Delto DG-3F-B。
- **采集/控制链路：** 文本任务 → 2D seed → RGB-D投影为3D点 → embodiment-conditioned flow matching → 6D抓取 → 仿真/真机执行。
- **自由度与传感：** 610个杂乱场景、334个物体、2.56M抓取位姿；不是人手关节数据。
- **反馈方式：** 抓取成功率，无触觉或抓力闭环。
- **对我们搭建平台的启发：** 保存 `language_instruction`、`target_object_id`、`seed_pixel`、`seed_point_3d`、`embodiment_id`、`grasp_pose_6d` 和 `execution_success`。
- **风险或不确定性：** 代码和数据仍处于占位状态；五指高DoF跨手迁移和真实接触安全性待验证。

### Masked Visual Actions

- **原始链接：** [arXiv](https://arxiv.org/abs/2607.19343) · [项目页](https://masked-visual-actions.github.io/) · [GitHub](https://github.com/HadiZayer/masked-visual-actions) · [Hugging Face](https://huggingface.co/HadiZayer/masked-visual-actions)
- **一句话概括：** 用视频中部分显示的机器人或物体运动表示动作，让视频模型补全其余世界变化。
- **硬件是什么：** 使用DROID、BEHAVIOR和自采真机数据；测试embodiment不出现在训练中。
- **采集/控制链路：** 初始图像 + masked action video + 文本 → Wan2.2 LoRA → 完整RGB rollout；显示机器人运动用于forward modeling，显示物体运动用于inverse modeling。
- **反馈方式：** 视觉结果预测，无触觉和真实力学反馈。
- **开源情况：** Apache-2.0；推理、LoRA训练recipe和权重已发布，DROID→URDF渲染工具仍待发布。
- **对我们搭建平台的启发：** 永久保存原始视频、对象mask和物体运动，新增 `mask_video`、`masked_entity_type`、`generated_rollout` 和 `checkpoint_hash`。
- **风险或不确定性：** 14B模型成本高，视觉合理不代表接触力学真实；精细接触和首帧不可见区域易产生伪影。

### ModPack DIY遥操作底座

- 不属于商业整机，但软件和主要硬件思路已经公开。
- 适合作为“手臂/底盘/主动视角”采集底座，与独立高DoF手套组合。
- 采购前应核算背包重量、Dynamixel主臂成本、Vision Pro、iPhone、移动机器人及F/T传感器成本。
- 对我们的近期价值高于直接购买整套全身遥操作系统，可优先复现日志与模块接口，再决定制造主臂。

### 执行手采购新增要求

HOST与EgoRecovery说明，产品SDK除实时控制外还应支持：

- 快速加载单个示教或技能参考；
- 保存任务进度、失败状态与恢复片段；
- 在策略接管、人工纠错和恢复模式之间无跳变切换；
- 导出命令、实际执行、触觉/电流以及失败前后完整时间窗。

## 6. 对我们项目的直接建议

1. **立即把失败恢复设为一级数据类型。** 不再把失败episode简单标记为无效，应保存失败起点、纠错时机、纠错幅度、恢复门控和最终结果。
2. **将ModPack加入“手臂/全身主设备”复现队列。** 先运行软件、分析统一logger和模块manifest，再决定制造6/7-DoF主臂。
3. **把手指采集与全身遥操作解耦。** ModPack负责手臂、底盘、头部和力反馈；现有22路编码器手套负责手指，可通过统一时钟和模块接口组合。
4. **新增任务进度与技能存储字段。** 参考HOST，保存任务视频、进度映射、技能检索key、未来观察目标和是否发生参数更新。
5. **建立人类恢复数据采集protocol。** 每项任务主动布置5–10类off-nominal状态，录制短恢复片段，并与少量真机恢复数据做成本等价比较。
6. **保留物体运动与像素动作。** Masked Visual Actions说明，普通视频中的对象mask和运动轨迹未来可用于跨本体world model训练。
7. **几何抓取与力控必须分层。** SeededGrasp生成候选位姿，V2F预测接触前抓力，执行手触觉/电流负责接触后修正。
