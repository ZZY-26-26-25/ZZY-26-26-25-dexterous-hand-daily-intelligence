## 2. 重点解读

### T-800: An 800 Hz Data Glove for Precise Hand Gesture Tracking

- **原始链接：** [Wiley 全文](https://onlinelibrary.wiley.com/doi/full/10.1002/smb2.70045) · [arXiv](https://arxiv.org/abs/2603.26403) · [作者页面](https://yzhu.io/publication/glove2026smartbot/)
- **一句话概括：** 通过 18 个分布式 IMU、广播式时间锁存和机械应力隔离，实现完整手部同步 800 Hz 采集，重点捕获传统 30–200 Hz 手套会遗漏的快速微动作。
- **硬件是什么：** 柔性可穿戴手套上布置 18 个 IMU 节点，每个节点包含传感、微控制和通信模块；结构采用机械应力隔离设计，降低穿戴弯折和局部受力对 IMU 的影响。
- **采集/控制链路：** 全局广播时间锁存 → 18 个节点同步采样 → 温度与空间标定 → 全手姿态重建 → 几何 retargeting → Shadow / Allegro / LEAP 关节目标。
- **自由度与传感：** 论文强调 18 节点全手惯性测量，而不是 18 个独立机器人关节命令。系统以 800 Hz 同步读取，避免分布式 IMU 时钟漂移。
- **反馈方式：** 仅采集人手运动，不向操作者提供触觉或力反馈。
- **方法核心与实验：** 论文通过快速手—物交互和转笔等动作证明，手部运动在 100 Hz 以上仍有显著能量；并将数据映射至 Shadow、Allegro、LEAP。文中报告 LEAP 与 Allegro 的指尖映射误差低于 Shadow，说明“仿人形态更接近”不等同于“可达空间更适合 retarget”。
- **和 DexUMI / DOGlove / UMI-Dex / SynGlove 的关系：** T-800 强调高频运动采集，不提供接触真值和反馈；DOGlove、SynGlove 更重视触觉/力反馈，UMI-Dex 更重视视频、腕部和执行端一致性。它适合作为这些系统的高频运动补充，而非完整替代。
- **对我们搭建平台的启发：** 当前 22 路编码器系统应区分“人体动作有效带宽”和“最终机器人控制频率”。即使机器人仅以 30–100 Hz 执行，原始编码器可尽量高频采样，并保留时间戳、滤波前数据和降采样版本。
- **风险或不确定性：** IMU 的绝对航向与长期漂移仍需外部参考；论文未开放完整硬件 BOM、PCB、固件和数据集；800 Hz 对供电、无线带宽、本地存储和多模态同步提出更高要求。

### LinkerHand L20Lite URDF 更新

- **原始链接：** [GitHub](https://github.com/linker-bot/linkerhand-urdf)
- **一句话概括：** 官方仓库近期更新 L20Lite 的命名、URDF 和 mesh，使数字模型与发布硬件保持一致。
- **硬件是什么：** LinkerHand 系列机器人手的数字模型仓库，当前覆盖 L20Lite/L10、L20、L25、O7、O6、L6 等型号及左右手。
- **采集/控制链路：** 人手/手套输入 → retargeting → URDF 运动学与关节限制 → LinkerHand 控制；URDF 同时影响仿真、碰撞检测、数据可视化和离线重放。
- **自由度与传感：** 本次是模型资产更新，不新增真实传感器接口。
- **反馈方式：** 无。
- **开源价值：** 提供模型、mesh 和左右手结构；近期提交明确说明 L20Lite 资产与硬件发布版对齐，并迁移部分 mesh 至 Git LFS。
- **和 DexUMI / DOGlove / UMI-Dex / SynGlove 的关系：** 这些人端采集方案都需要稳定的目标手 URDF。目标 URDF 改版后，即使原始人手数据不变，retarget 结果也可能变化，因此必须保存 URDF 版本与 commit SHA。
- **对我们搭建平台的启发：** 每条 episode 增加 `robot_urdf_repo`、`robot_urdf_commit`、`mesh_version` 和 `joint_mapping_version`，避免模型更新后无法复现旧数据。
- **风险或不确定性：** README 中 L10 行仍指向 `l20lite`，命名迁移可能尚未完全收敛；URDF 与真实控制接口、触觉布局和电机参数是否完全一致仍需实机校验。

### Inspire Hand Retargeting

- **原始链接：** [GitHub](https://github.com/namaewa-im/inspire-hand-retargeting)
- **一句话概括：** 使用 RealSense D405 与 MediaPipe 直接输出可经 UDP 传输的 6 维手部命令，是一个轻量、低成本、容易改造的视觉 retargeting 原型。
- **硬件是什么：** Intel RealSense D405，相机只需输出彩色流；核心估计使用 MediaPipe Hand Landmarker。
- **采集/控制链路：** D405 RGB → MediaPipe 21 landmarks → 6 个归一化关节量 → jitter filter → PySide6/pyqtgraph UI → UDP。
- **自由度与传感：** 输出拇指 CMC yaw、CMC pitch，以及食指、中指、无名指、小指 MCP，共 6 维；不输出 PIP/DIP，也不直接输出手腕全局位姿。
- **反馈方式：** 无机器人端状态、无触觉和力反馈。
- **开源内容：** 包含关节估计、实时 UI、滤波、UDP subscriber、相机与 MediaPipe 测试代码。
- **和 DexUMI / DOGlove / UMI-Dex / SynGlove 的关系：** 成本显著低于外骨骼和数据手套，但可观测自由度和遮挡鲁棒性也更低。适合作为 Inspire Hand 低自由度命令输入、教学 demo 或备用通道，不适合作为高保真全手 ground truth。
- **对我们搭建平台的启发：** 可以把它作为外骨骼系统的视觉冗余通道，用于检测编码器异常、手套脱落和粗姿态一致性，而不是替代 22 路编码器。
- **风险或不确定性：** README 明确标记为实验代码；输出为归一化分数而非人体解剖角度；D405 近距离遮挡、左右手识别和不同手型标定仍需测试；未见明确许可证。

### Dexterous Slip Detection for XHAND1

- **原始链接：** [GitHub](https://github.com/sumriver/dexterous-slip-detection)
- **一句话概括：** 以物理能量流为抽象，在 MuJoCo 中为 XHAND1 构建触觉滑移检测、腕部/抓力控制和后续在线力优化路线。
- **硬件是什么：** 目标真机为 XHAND1 / XHAND1 Pro；当前主要在 MuJoCo 中使用 XHAND1 和 Shadow Hand 模型。
- **采集/控制链路：** 仿真 contact/taxel → 触觉能量状态 → 滑移判定 → grip/wrist action → 后续 LGM-FF 在线学习和 pMPC 力优化。
- **自由度与传感：** 依赖 XHAND1 触觉阵列和机器人状态；近期代码加入腕部非零动作与抓力 teacher search，并批量生成通过的 open-loop 样本。
- **反馈方式：** 机器人端滑移与接触能量反馈。
- **开源内容：** Apache-2.0；含 MuJoCo 场景、XHAND1/Shadow 对照、能量流模块、实验脚本、测试和详细中文研究文档。
- **和 DexUMI / DOGlove / UMI-Dex / SynGlove 的关系：** 人端方案解决“怎么示教”，该仓库解决机器人执行时“是否开始滑、应怎样增大抓力或调整腕部”。可以作为示教后执行端安全层。
- **对我们搭建平台的启发：** 数据应增加 `slip_score`、`energy_flow_state`、`grip_residual`、`wrist_residual`、`taxel_mapping_version` 和 `teacher_search_result`。
- **风险或不确定性：** 当前 XHAND1 抓瓶仍需调参；能量流法尚未进入真机；仓库路线中的在线学习与 pMPC 仍待实现，不能将框架完成等同于实际防滑成功。

## 6. 对我们项目的直接建议

1. **重新评估 22 路编码器的采样链路。** 机器人控制可以是 50–100 Hz，但原始传感建议尽可能达到 200 Hz 以上，并评估局部高速动作是否需要 500–800 Hz burst mode。T-800 证明快速操作存在明显 >100 Hz 动态。
2. **采用统一硬件时间戳，而不是依赖 PC 到达时间。** 每块 ADC/MCU 应由统一时钟或广播锁存对齐；否则 22 路编码器、IMU、LRA、视频和腕部 6D 在高频动作中会产生不可恢复的相位误差。
3. **保存目标手模型版本。** LinkerHand 本轮 URDF 更新说明，数据集中必须记录 URDF 仓库、commit、mesh 版本和关节映射版本。
4. **加入视觉冗余输入。** 可参考 Inspire Hand Retargeting，用 D405/广角相机和 MediaPipe 输出粗姿态，作为编码器手套的异常检测和快速演示通道。
5. **提前设计滑移字段。** 即使当前手套只采人手动作，未来机器人端也应记录 `slip_state`、`slip_score`、`grip_residual`、`wrist_residual` 和 taxel 空间布局。
6. **建立多手型 URDF 注册表。** 可整合 LinkerHand 与 WHATsLAB 资产，对每种目标手维护 `embodiment_id`、关节定义、可达空间、耦合关系和 License。
7. **不要把新仓库等同于成熟产品。** Inspire 和 XHAND1 slip 项目均有工程价值，但仍处于实验/仿真阶段，应先做代码审阅和 smoke test，再决定采购或硬件路线。
