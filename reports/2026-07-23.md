## 2. 重点解读

### Wuji Hand 2 Firmware v2.1.0 / SDK v2026.7.21

- **原始链接：** [官方更新日志](https://docs.wuji.tech/docs/en/wuji-hand/latest/release-notes/) · [PyPI](https://pypi.org/project/wuji-sdk/)
- **一句话概括：** Wuji Hand 2 第一次将五根指尖的触觉数据作为正式 SDK 接口发布，并同时解决触觉标定、流时间戳漂移和通信故障安全停机问题。
- **硬件是什么：** Wuji Hand 2 Beta 五指灵巧手；SDK 支持 Linux x86-64 与 ARM64，并提供 Python 3.10–3.14 二进制包。
- **采集/控制链路：** SDK 首先读取 `FingertipSensorInfo` 元数据，再按元数据解码每帧 `FingertipSensorData`；可查询每根手指的传感器型号和校准状态，并用 `tactile_calibrate()` 在无载状态下重置五指零点。
- **自由度与传感：** Wuji Hand 2 为 20-DoF 五指手；本次更新重点是每指触觉流、关节状态、编码器健康监控和诊断信息。
- **反馈方式：** 机器人端触觉；没有证据表明 Wuji Glove 提供主动力反馈。
- **开源/产品状态：** SDK wheel 可下载，发布带有 Sigstore provenance；底层开发仓库仍为私有，因此属于“公开接口与二进制 SDK”，不是完整源码开放。
- **和 DexUMI / DOGlove / UMI-Dex / SynGlove 的关系：** 人端采集方案负责输出动作；Wuji 更新补齐了目标手执行端的自描述触觉协议，使手套动作、机器人关节和触觉可以在同一 episode 中可靠记录。
- **对我们搭建平台的启发：** 触觉帧必须携带 `sensor_metadata_version`、`finger_id`、`layout`、`unit`、`calibration_state` 和 `zero_timestamp`，不能只保存一段无说明数组。还应保留设备时钟与主机时钟重对齐事件。
- **风险或不确定性：** Beta 版本参数可能继续变化；未公开触觉频率、量程、迟滞和标定误差；固件与 SDK 必须成套升级。

### MVP-Tac

- **原始链接：** [arXiv](https://arxiv.org/abs/2607.18660) · [项目页](https://mvp-tac.github.io/)
- **一句话概括：** 在一个微型传感头中，用同一相机在环境视觉和光弹性触觉之间切换，从而兼顾观察和轻力触诊。
- **硬件是什么：** 由保护层、半透明反射膜、VytaFlex 20 光弹性层、亚克力间隔件、圆偏振片、微型内窥 CMOS 相机和 LED 构成；相机为 1280×720、76°视场。
- **采集/控制链路：** 外部光强较高、内部 LED 关闭时工作在视觉模式；LED 打开后通过反射式圆偏振光路采集受力干涉条纹；图像和光弹性梯度指标共同用于力估计与硬度分类。
- **自由度与传感：** 不涉及机器人手 DoF；力标定范围 0–2 N，传感端直径约 7 mm 级，组件与模具成本低于 20 美元。
- **反馈方式：** 机器人/工具端视觉和触觉；不向操作者主动反馈。
- **方法核心与实验：** 对约 2500 组图像—力数据做标定；在组织仿体上实现暴露肿瘤 97%、皮下肿瘤 92% 分类准确率，并完成模拟结肠镜的视觉三维映射和原位触诊。
- **和现有方案的关系：** 与 GelSight/DIGIT 相比，它不依赖复杂结构光和不透明接触层；与 OptiTac 等远置光学方案相比，它保留了同轴环境视觉。
- **对我们搭建平台的启发：** 可作为单指触觉原型、狭小空间探针或腕部辅助相机的候选；数据中应分开保存 `visual_mode_frame`、`tactile_mode_frame`、`illumination_state`、`force_ground_truth` 和 `sensor_build_id`。
- **风险或不确定性：** 当前面向轻力触诊，尚未验证多方向剪切力、快速滑移、长期磨损和灵巧手高速操作；视觉与触觉不是严格同时曝光，而是切换模式。

### UHAS：Unified Hand Action Space

- **原始链接：** [项目页](https://irvlutd.github.io/UHAS/) · [GitHub](https://github.com/IRVLUTD/UHAS_sim) · [arXiv](https://arxiv.org/abs/2607.03570)
- **一句话概括：** 将不同手的动作统一表示为一个规范球面的形变，再通过 Cascade IK 转成具体手的关节命令。
- **硬件是什么：** 支持 LEAP Hand、Allegro Hand、Shadow Hand 和 MANO Human Hand，并允许从任意 URDF 自动生成球面表示。
- **采集/控制链路：** 目标手 URDF → 自动构建规范球面和表面对应 → 策略输出球面形变 → CIK 转为目标手关节 → Isaac Lab 或真机执行。
- **自由度与传感：** 统一动作与观测不绑定具体关节数量；CIK 运行最高约 150 Hz。当前重点是本体状态和几何动作，没有触觉。
- **开源内容：** Apache-2.0；包含 Isaac Sim/Isaac Lab 环境、URDF 处理工具、多手联合训练、预训练模型、未知手接入文档和真机部署入口。
- **实验：** 在 LEAP 和 Allegro 上完成方块手内重定向；多手联合策略可迁移到未参与训练的手，尽管零样本性能仍低于目标手专训策略。
- **和现有方案的关系：** 与 AnyDexRT、传统 dex-retargeting 主要解决“人手到某只手”不同，UHAS 更关注“一个策略控制多种机器人手”。
- **对我们搭建平台的启发：** 应将 `human_action_space`、`unified_action_space`、`robot_joint_space` 分开保存；现有 22-DoF 外骨骼数据可先映射到 UHAS，再分别下发到 LEAP、Allegro、Wuji 或 MIDAS。
- **风险或不确定性：** 当前任务主要是仿真与方块重定向；不含触觉、力控和复杂工具使用；Isaac Sim 4.5/Isaac Lab 2.2.1 环境依赖较重。

### T-800: An 800 Hz Data Glove

- **原始链接：** [Wiley](https://onlinelibrary.wiley.com/doi/10.1002/smb2.70045) · [arXiv](https://arxiv.org/abs/2603.26403) · [项目页](https://yzhu.io/publication/glove2026smartbot/)
- **一句话概括：** 18 个分布式 IMU 通过广播式同步实现完整手部 800 Hz 采样，重点捕获普通数据手套遗漏的高频动作成分。
- **硬件是什么：** 柔性手套上布置 18 个惯性测量节点，每个节点含传感、微控制和通信部分，并通过结构设计减少手套应力对 IMU 的影响。
- **采集/控制链路：** 广播时间锁存 → 多节点同步采样 → 温度/空间标定 → 手部姿态重建 → Shadow/Allegro/LEAP retargeting。
- **反馈方式：** 无主动触觉或力反馈。
- **对我们搭建平台的启发：** 控制频率可以低于原始采样频率。建议编码器/IMU以 200 Hz 以上常规采样，并预留 500–800 Hz burst mode，同时保存原始、滤波和降采样三套数据。
- **风险或不确定性：** IMU 航向漂移仍需外部参考；800 Hz 会增加功耗、带宽、同步和存储压力；硬件 BOM、PCB 和固件尚未完整公开。

### LinkerHand / Inspire / XHAND1 工程更新

- **原始链接：** [LinkerHand URDF](https://github.com/linker-bot/linkerhand-urdf) · [Inspire Retargeting](https://github.com/namaewa-im/inspire-hand-retargeting) · [XHAND1 Slip](https://github.com/sumriver/dexterous-slip-detection)
- **一句话概括：** 三项更新分别解决目标手模型版本、低成本视觉输入和机器人执行端滑移闭环。
- **关键关系：** LinkerHand 模型更新后必须重新确认关节映射和碰撞；Inspire 视觉管线适合作为编码器手套冗余通道；XHAND1 项目适合定义滑移、抓力残差和腕部残差字段。
- **对我们搭建平台的启发：** 每个 episode 保存 `urdf_commit`、`retargeter_version`、`vision_pose_estimate`、`slip_score`、`grip_residual` 和 `wrist_residual`。
- **风险或不确定性：** Inspire 项目仍是实验代码；XHAND1 当前主要为 MuJoCo；URDF 不等同于真实动力学和触觉模型。

## 6. 对我们项目的直接建议

1. **优先审阅 Wuji v2026.7.21 触觉接口。** 将自描述元数据机制写入我们的通用触觉 schema，避免不同厂商 taxel 数量和布局变化时破坏数据兼容。
2. **复现一个 MVP-Tac 原型。** 先验证 0–2 N 法向力、视觉/触觉切换和材料耐久，再判断是否适合安装在指尖、工具或腕部探针。
3. **拉取 UHAS 做跨手型冒烟测试。** 用 LEAP/Allegro 现成模型跑通后，尝试导入 MIDAS、Wuji 或 LinkerHand URDF；评估统一动作空间是否保留拇指和侧摆细节。
4. **将传感频率与控制频率彻底分离。** 原始编码器/IMU保留高频数据，机器人执行端按硬件能力降采样；所有派生数据必须保留父数据时间戳。
5. **建立手模型版本注册表。** 保存 `urdf_repo`、`urdf_commit`、`mesh_hash`、`joint_mapping_version`、`collision_version` 和 `license`。
6. **加入视觉冗余姿态。** D405/MediaPipe 可用于粗姿态校验、异常检测和无手套快速 demo，但不能替代编码器 ground truth。
7. **提前定义执行端触觉事件。** 增加 `slip_state`、`slip_score`、`grip_residual`、`wrist_residual`、`tactile_zero_event` 和 `sensor_fault`。
