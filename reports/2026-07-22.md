# 灵巧手每日情报｜2026-07-22

> 本报告采用 Awesome-UMI 风格条目化结构，保留简介、来源、硬件、传感、开源状态、研究价值和项目启发。

## 0. 今日一句话判断

今日重点集中在三个方向：

1. **力觉成为 VLA 长期记忆的重要模态**：FM-VLA 表明长时间 wrench 历史能够补充视觉无法观察的接触事件。
2. **开放灵巧手硬件继续成熟**：Handroid 提供低成本、多自由度、可扩展硬件平台。
3. **商业灵巧手进入数据闭环竞争**：AGIBOT OmniHand 3 Ultra-M、GenRobot DAS Dex 展示“机器人手本体 + 人类数据采集 + 模型训练”的完整路线。

对我们的灵巧手数据采集平台最重要的结论：未来数据格式不能只保存关节角，而必须保存：

- human command
- retarget command
- execution state
- motor current
- wrench
- tactile
- timestamp
- embodiment version

# 2. 重点解读

## FM-VLA

### 原始链接

https://arxiv.org

### 一句话概括

利用长期腕部力觉历史作为VLA记忆，使机器人能够理解视觉不可见的接触状态。

### 硬件

- AGIBOT G1 双臂机器人
- 双7DoF机械臂
- 双腕六维力/力矩传感器
- 多视角RGB相机

### 数据链路

机器人状态 + RGB + wrench history
→ force memory encoder
→ VLA action expert
→ robot action

### 传感

- 100Hz wrench
- joint state
- gripper state
- RGB

### 反馈

机器人端力觉，不包含人端主动反馈。

### 实验

任务包括：

- 隐藏物体寻找
- 精确按键
- 指定次数擦拭

结果显示力觉记忆明显优于视觉历史。

### 对项目启发

数据采集系统必须保留长时间力觉序列，而不是只保存当前step。

新增字段：

```
wrench_history
contact_event
memory_window
force_encoder_version
```

## Handroid

### 原始链接

https://github.com

### 一句话概括

开放式低成本多自由度灵巧手，可作为研究级机器人手平台。

### 硬件

- 20DoF手形态
- 27DoF人形形态
- Dynamixel执行器
- Vision Pro遥操作

### 传感

包含：

- joint position
- velocity
- load
- temperature
- IMU

暂未集成高密度触觉。

### 开源

包含：

- CAD
- BOM
- 软件接口
- 仿真环境

### 与项目关系

适合作为：

外骨骼手套 → 灵巧手执行端

的开放测试平台。

# 6. 对项目建议

1. 建立统一 embodiment schema。
2. 保留command与execution差异。
3. 保存原始触觉，不只保存latent。
4. 将UMI、手套、本体三类数据统一管理。
5. 采购设备必须要求SDK和原始数据权限。
